# Final Experiment Claims And Limitations

| Claim | Evidence | Limitation | Safe wording |
|---|---|---|---|
| Fixed-capacity 3v2-to-5v4 zero-shot transfer is completed. | Train/eval pipeline uses 3v2 training and 5v4 evaluation without fine-tuning. Final 5v4: `red_win_rate=0.93`, `mav_survival_rate=0.99`, `red_missile_hits_mean=2.38`. | This is not arbitrary-size generalization. It relies on fixed `max_red=5`, `max_blue=4` padding/masks. | The method demonstrates fixed-capacity zero-shot scale transfer from 3v2 to 5v4. |
| Unified V2 observation supports 3v2 and 5v4 with one actor schema. | Actor obs dim is `96`; critic state dim is `480`; padding and valid/alive/observed masks support missing slots and added UAVs. | It is BRMA-inspired but not a full BRMA attention encoder. | We use a BRMA-inspired entity/mask observation with fixed-capacity padding. |
| Heterogeneous actor structure is implemented. | `red_0` uses the MAV actor; all red UAVs share one UAV actor in both 3v2 and 5v4. | This is HAPPO reference v0, not full TAM-HAPPO with GRU and attention. | We implement a role-separated actor structure with independent MAV and shared UAV actors. |
| UAV attack behavior transfers to 5v4. | Final 5v4: `red_missiles_fired_mean=2.59`, `red_missile_hits_mean=2.38`, `blue_dead_mean=2.33`. | Wins are mostly timeout alive-advantage, not all elimination wins. | The learned UAV attack behavior remains active in the larger 5v4 zero-shot scenario. |
| MAV survival is high in the final 5v4 setting. | Final 5v4: `mav_survival_rate=0.99`. | MAV trajectory is not a strict rear-support orbit. | The MAV remains alive in most 5v4 episodes, but its learned trajectory is best described as forward-survival / loose support. |
| Geometry curriculum is necessary for normal combat behavior. | Direct normal attempt: `red_fire=0.05`, `red_hit=0.00`. Geometry curriculum normal best: `red_fire=1.82`, `red_hit=1.56`, `blue_dead=1.52`. | Latest checkpoint can still collapse; best checkpoint selection is required. | Easy-to-medium-to-normal curriculum recovers a usable normal-geometry checkpoint. |
| F-16 MAV surrogate is a trainability choice. | F-22 branch was unstable under the current high-level action + PID path; F-16 surrogate enabled stable HAPPO validation. | It is not a faithful physical reproduction of an F-22 MAV. | We use an F-16 dynamics surrogate for the MAV while visualizing `red_0` as MAV/F22 in ACMI. |
| Wrapped-heading oracle imitation is needed. | Plain heading MSE failed for circular heading actions; wrapped heading loss produced easy-combat firing and enabled curriculum. | This is an imitation/prior mechanism, not a new environment capability. | The oracle imitation uses wrapped heading error to respect the circular action dimension. |
| Blue can target the MAV by code path. | Blue target selection scans all alive red slots, including `red_0`; missile launch scans `self.red_planes`. | In the audited 20 5v4 episodes, blue missiles targeted UAVs, not MAV. | Blue is not prevented from selecting MAV, but actual audited missile launches targeted red UAVs. |
| 5v4 metrics remain valid despite MAV trajectory limitations. | Final 5v4 metrics pass the success gate; behavior audit documents MAV role behavior as not strictly rear-support. | The MAV behavior should not be described as fully paper-aligned. | The zero-shot metrics are valid, with a documented behavior-level limitation on MAV trajectory consistency. |
| Full BRMA-MAPPO is not reproduced. | No biased random masked attention or BRMA entity attention encoder is implemented. | The observation is inspired by BRMA, not a full algorithm reproduction. | We compare against BRMA-inspired design choices, not a full BRMA-MAPPO reproduction. |
| Full TAM-HAPPO is not reproduced. | No GRU, temporal module, or attention-enhanced value network is implemented. | HAPPO reference v0 is a simplified role-separated baseline. | The method is a HAPPO-style reference implementation, not full TAM-HAPPO. |
| Combat zero-shot transfer is not solved in the strongest possible sense. | 5v4 has strong red fire/hit and high survival, but only `red_elimination_win_rate=0.14`; many wins are timeout alive advantage. | More aggressive elimination behavior and stricter MAV support behavior remain future work. | The final result demonstrates meaningful 5v4 attack-transfer indicators, not a complete solution to all combat transfer behaviors. |
